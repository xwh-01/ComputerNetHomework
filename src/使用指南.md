# OptTrans 使用指南

## 项目简介

OptTrans 会把二进制数据编码成码图，再封装成视频来传输。当前项目使用 `V2` 编码规则，优化了定位稳定性和解码表现。

核心特性：
- 单帧有效载荷为 `1384` 字节
- 支持 `START / END` 标记帧
- 解码时自动处理透视变换和模糊
- 支持限制编码时长和保存截断数据
- 提供逐位比较工具验证解码正确性

## 环境要求

- Python 3.12 或更高版本
- 建议使用虚拟环境

安装依赖：

```powershell
py -3.12 -m pip install -r requirements.txt
```

## 快速开始

### 1. 编码文件为视频

命令格式：

```powershell
py -3.12 .\opttrans_video.py encode <输入文件> <输出视频> [参数]
```

可选参数：
- `--fps`：视频帧率，默认 `7`
- `--marker-frames`：开始和结束标记各重复多少帧，默认 `8`，设为 `0` 时不添加标记
- `--data-frames`：每张数据码图重复多少帧，默认 `2`
- `--max-seconds`：最大编码时长（秒），只编码指定时间的数据
- `--save-cut`：保存截断后的二进制文件为 `*_cut.bin`

示例 1：基本编码
```powershell
py -3.12 .\opttrans_video.py encode .\examples\01.bin .\examples\output.avi
```

示例 2：限制 1 秒编码，保存截断数据
```powershell
py -3.12 .\opttrans_video.py encode .\examples\01.bin .\examples\output.avi --fps 20 --marker-frames 0 --data-frames 2 --max-seconds 1 --save-cut
```

输出：
```
Video encode succeeded: 10 data frames written to .\examples\output.avi
Encoded 13840 bytes at 110.72 kbps
```

### 2. 解码视频为文件

命令格式：

```powershell
py -3.12 .\opttrans_video.py decode <输入视频> <输出文件>
```

示例：
```powershell
py -3.12 .\opttrans_video.py decode .\examples\output.avi .\examples\01_decoded.bin
```

### 3. 比较文件并生成验证文件

命令格式：

```powershell
py -3.12 .\compare_bits.py <文件1> <文件2>
```

示例：
```powershell
py -3.12 .\compare_bits.py .\examples\01_cut.bin .\examples\01_decoded.bin
```

输出：
```
✓ Comparison complete!
  File 1: 01_cut.bin (13840 bytes)
  File 2: 01_decoded.bin (13840 bytes)
  Total bits: 110720
  Correct bits: 110720
  Accuracy: 100.00%
  Val file saved: 01_cut.val
```

## 技术说明

### 编码原理
1. 将二进制文件按 `1384` 字节分帧
2. 每帧添加 Reed-Solomon 纠错码
3. 生成二维码样式的码图
4. 封装成视频（可选添加 START/END 标记）

### 解码原理
1. 检测视频中的码图
2. 自动纠正透视变换和模糊
3. 解码数据并校验纠错码
4. 按帧顺序拼接数据

### 传输速率计算
- 有效帧率 = FPS / data-frames
- 每秒数据 = 有效帧率 × 1384 字节
- kbps = (每秒数据 × 8) / 1000

## 使用建议
- 编码时根据需要设置 `--max-seconds` 控制数据量
- 解码时确保视频完整，不要裁剪头尾
- 拍照/录制时保持码图清晰，避免反光和模糊
- 使用 `--save-cut` 和 `compare_bits.py` 验证解码正确性

## 常见问题

### 1. 编码失败
- 检查输入文件是否存在
- 确认 Python 版本和依赖安装正确

### 2. 解码失败
- 检查视频是否完整包含码图
- 尝试增加 `--data-frames` 提高解码成功率
- 确保视频没有被过度压缩或裁剪

### 3. 比较文件不匹配
- 检查视频编码和解码过程是否完整
- 确认 `--max-seconds` 设置是否一致
- 检查录制质量是否足够清晰
